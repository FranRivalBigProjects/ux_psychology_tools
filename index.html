Cognitive Load & UX Psychology FAQ

What is cognitive load in UX design? ▼
Cognitive load is the amount of mental effort required to use an interface. High cognitive load means users must think hard to complete tasks, leading to confusion, slower performance, and abandonment. Low cognitive load means users can navigate intuitively with minimal mental effort. The goal of UX design is to reduce cognitive load through clear information architecture, familiar patterns, and progressive disclosure of information.
How does cognitive load relate to decision fatigue? ▼
Decision fatigue occurs when users must make too many choices in sequence, leading to poor decisions or choice paralysis. It's related to cognitive load but distinct. You can have low cognitive load but high decision fatigue (too many simple choices), or high cognitive load but few decisions (one complex task). Both metrics are important for UX design—minimize both to create smooth user experiences.
What score should I aim for? ▼
It depends on your audience and interface type. Consumer-facing products should aim for 20-35 (low cognitive load). Professional tools can tolerate 40-55 (moderate load) if users are trained. Technical documentation can go up to 60+ (high load) for expert audiences. General rule: lower is better, but context matters. Test with real users to validate your score.

What is Fitts's Law and Hick's Law? ▼
Fitts's Law states that the time to reach a target is proportional to the distance and inversely proportional to the size. In UI terms: make buttons larger or put them closer to where users need them. Hick's Law states that the time to make a decision increases with the number of choices. In UI terms: reduce menu items, simplify choice architecture. Both principles help reduce cognitive load and speed up user interactions.
Should I use the same cognitive load for all user types? ▼
No. Expert users (like software developers or traders) can handle higher cognitive load than novices. Design for your specific audience. For consumer products, assume low expertise and aim for low cognitive load (20-35). For professional tools, you can increase it (40-55) if your users are trained. Always validate assumptions with user testing.
